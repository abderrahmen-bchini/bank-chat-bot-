1. Subnets needed

We need separate subnets for web and database, and high availability.

Each subnet must have 256 IPv4 addresses → /24.

VPC CIDR: 10.0.0.0/16

Subnets:

Public Subnet 1 (Web – AZ1): 10.0.0.0/24

Public Subnet 2 (Web – AZ2): 10.0.1.0/24

Private Subnet 1 (Database – AZ1): 10.0.2.0/24

Private Subnet 2 (Database – AZ2): 10.0.3.0/24

This gives:

Separation of web and database

High availability (multiple AZs)

2. How the VPC is networked

Create one VPC with CIDR 10.0.0.0/16

Attach an Internet Gateway (IGW) to the VPC

Deploy:

Web servers in public subnets

Database servers in private subnets

Create a NAT Gateway in a public subnet

Private subnets use the NAT Gateway for internet access (patch updates)

4. Do you need Security Groups? Justify.

Yes, Security Groups are required

Security Groups act as a custom firewall layer, which is explicitly required.

Default configuration is NOT sufficient

Why customize?

Default SG allows all inbound from itself (not secure)

We need tight access control between web and database

Principle of least privilege

5. Security Group rules (customized)

Web Server Security Group

Inbound rules:

HTTP (80) → 0.0.0.0/0

HTTPS (443) → 0.0.0.0/0

SSH (22) → Admin IP only

Outbound rules:

Allow all outbound traffic

Database Server Security Group

Inbound rules:

Database port (e.g., 3306 for MySQL)

Source: Web Server Security Group

SSH (22)

Source: Admin IP only

Outbound rules:

Allow all outbound traffic (for updates via NAT Gateway)

6. Do you need Network ACLs? Justify.

Not strictly required, but recommended

Why?

Security Groups already protect instances

ACLs add an extra security layer at subnet level

Useful for defense-in-depth in cloud environments

So:

Optional but good practice

7. Network ACL rules (if configured)

Public Subnet ACL

Inbound:

Allow HTTP (80) from 0.0.0.0/0

Allow HTTPS (443) from 0.0.0.0/0

Allow ephemeral ports (1024–65535)

Deny all other traffic

Outbound:

Allow all traffic

Private Subnet ACL

Inbound:

Allow database port from public subnets only

Allow ephemeral ports from NAT Gateway

Deny all other inbound traffic

Outbound:

Allow outbound traffic to NAT Gateway

Deny direct internet access
