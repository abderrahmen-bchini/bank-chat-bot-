## Using Retrospectives

Create a GitHub Project with columns for retrospective organization:

### Retrospective Project Columns
- **No Status** — Unfinished action items
- **TODO** — Retrospective agenda items
- **Done** — Completed work/positive outcomes
- **Suggestion & Stuck** — Improvement suggestions and blockers
- **Action Items** — Items to improve moving forward

### Process
1. Hold sprint retrospective meeting
2. Document findings in RETROSPECTIVE.md
3. Create GitHub issues for action items
4. Organize issues in project board by category
5. Assign owners and track progress
